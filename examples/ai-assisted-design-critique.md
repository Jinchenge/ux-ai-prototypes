# AI-Assisted Design Critique Example

This example shows how a UX designer might use AI to support, but not replace, design critique.

## Scenario

A product team is reviewing an onboarding flow for a new AI-powered workspace tool.

## Design Critique Prompt

```text
Act as a UX design reviewer. Review the following screen description and provide feedback on clarity, hierarchy, user control, accessibility, and possible friction points. Separate observations, risks, and suggestions.

Screen description:
[Describe the screen or flow here]
```

## Example Review Structure

### Observations

- The primary action is visible but competes with secondary text.
- The user may not understand what data is required before continuing.
- The screen introduces several concepts at once.

### Risks

- Users may continue without understanding how their data will be used.
- The interface may feel more technical than necessary for first-time users.

### Suggestions

- Add a shorter explanation near the primary action.
- Move detailed technical information to a secondary disclosure.
- Use clearer button language that matches the user's goal.

## Human Review Notes

AI feedback should be checked by a designer before it becomes a product recommendation. The designer should compare the feedback against user research, product goals, and accessibility requirements.
