# Product Copy Experiment

This example explores how AI can help generate and compare UX writing options.

## Scenario

A product needs an empty state message for a dashboard with no saved projects.

## Prompt

```text
Generate five empty state messages for a UX design tool. The tone should be clear, calm, and helpful. Include a short title, one sentence of supporting text, and a primary action label.
```

## Evaluation Criteria

- Is the message clear?
- Does it explain what happened?
- Does it guide the user toward the next step?
- Is the tone appropriate for a productivity tool?
- Is the action label specific?

## Example Output Format

| Option | Title | Body | Action |
| --- | --- | --- | --- |
| 1 | No projects yet | Create your first project to start organizing your design work. | Create project |
| 2 | Start with a new project | Projects help you keep research, flows, and notes in one place. | New project |

## Designer Notes

AI can generate useful variations quickly, but the final copy should be selected based on product context, user expectations, and brand voice.
